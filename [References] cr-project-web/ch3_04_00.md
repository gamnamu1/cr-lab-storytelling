# **10-1. 사용자 경험 흐름 설계** 
## **10-1-1. 접속 및 기사 입력** 
### **사용자가 웹앱에 접속**
- 직관적인 메인 화면에 "기사 분석하기" 대형 버튼 표시 
- 로그인/비로그인 모두 사용 가능 (로그인 시 분석 히스토리 저장) 
- PC/모바일/태블릿 모두 반응형 지원 
### **기사 수집 방식 선택**
- **URL 입력** (권장 방식): 기사 URL을 입력하면 시스템이 자동으로 내용 추출 및 불필요 요소 필터링 
- **텍스트 직접 입력** : 기사 본문을 복사하여 텍스트 상자에 붙여넣기 가능 
- **파일 업로드** : PDF, DOCX, TXT 형식 지원 (OCR 자동 변환) 
### **추가 정보 입력(선택사항)**
- 언론사, 발행일, 기자명 등 메타데이터 자동 추출 및 확인 과정 
- 사용자 수정 가능 (자동 추출 실패 시) 
### **기사 콘텐츠 추출 및 무결성 확인 시스템**
- **적응형 크롤링 시스템** : 국내 주요 언론사 약 50개의 HTML 구조 분석 기반 맞춤형 추출 
- **콘텐츠 정제 파이프라인** : 광고, 관련 기사 링크 등 본문과 무관한 요소 자동 필터링 
- **콘텐츠 완전성 검사** : 기사 구조 분석으로 헤드라인, 본문, 인용문 등 필수 요소 누락 감지 
- **원본 참조 시스템** : 텍스트 직접 입력 시에도 원본 URL 제공 권장 및 차이점 감지 
- **오류 복원 메커니즘** : 크롤링 실패 시 Readability 알고리즘 등 대체 경로 활용 
### **법적 보호 장치**
- **면책 조항 자동 첨부** : "이 평가는 제출된 내용만을 기준으로 하며, 원본 기사의 전체 맥락을 고려하지 못할 수 있습니다" 명시 
- **이의제기 프로세스** : 언론사/기자가 불공정한 평가라고 판단할 경우 간편한 이의제기 절차 제공 
- **사용자 주의 안내** : "공정한 평가를 위해 기사 전문을 누락 없이 입력해주세요" 등 가이드라인 제공 
## **10-1-2. 기사 유형 분석 및 평가 준비** 
### **기사 유형 자동 감지**
- AI가 제목과 본문 패턴 분석하여 기사 유형 판별 
- 스트레이트 뉴스, 해설 기사, 칼럼, 인터뷰 등 구분 
- 분석 가능 여부 판단: 평가 대상은 정치/사회 분야 스트레이트 뉴스로 제한 
### **기사 전처리 수행**
- HTML 태그 제거, 광고 내용 필터링, 텍스트 정제 
- 문단 및 문장 구조 분석, 인용문/통계 데이터 구분 
- 메타데이터 추출 (작성일, 언론사, 기자명 등) 
### **분석 불가 기사 처리**
- 분석 대상이 아닌 경우 (예: 칼럼, 광고성 기사) 분석 중단 
- "이 유형의 기사는 현재 분석이 지원되지 않습니다" 안내 메시지 
- 대안 제시: "정치/사회 분야의 스트레이트 뉴스만 분석 가능합니다" 
## **10-1-3. 다차원 품질 분석 실행** 
### **8개 핵심 평가 차원 기반 분석**
- 진실성과 정확성: 사실 검증, 출처 명확성 
- 투명성과 책임성: 취재원 투명성, 익명 취재원 사용 적절성 
- 균형성과 공정성: 관점 다양성, 편향성 여부 
- 독립성과 자율성: 상업적 이해관계 독립 여부 
- 인권과 프라이버시: 개인정보 보호, 인권침해 요소 
- 전문성과 심층성: 맥락 제공, 배경 설명 충실도 
- 언어와 표현의 윤리: 과장/왜곡 표현, 자극적 용어 
- 디지털 환경의 윤리: 클릭베이트 요소, 데이터 활용 윤리 
### **다차원 매트릭스 평가 방식 적용**
- 각 문제 패턴이 다양한 평가 차원에 미치는 영향 계산 
- 예: 무주체 피동형 표현("~로 알려졌다")이 투명성(-20점), 진실성(-15점)에 미치는 영향 계산 
- 30개 이상의 문제 패턴에 대한 종합적 분석 수행 
### **텍스트 마이닝 및 자연어 처리**
- 무주체 피동형 표현 탐지 ("~로 알려졌다", "~로 전해졌다") 
- 익명 취재원 사용 빈도 및 적절성 평가 
- 주관적 형용사/부사, 감정적 표현 사용 분석 
- 인용구의 주관적 술어 사용 평가 

# **10-2. 결과 전달 및 시각화 시스템** 
## **10-2-1. 계층적 평가 결과 구조** 
### **최종 평가 리포트 핵심 구성요소**
평가 리포트는 단순한 점수 나열이 아닌, 체계적이고 교육적으로 구성할 필요. 다음 요소들이 균형 있게 포함되어야 한다: 

- **기본 메타 정보** 
  - 기사 제목 및 원문 링크 
  - 언론사명/기자명/발행일자 
  - 기사 유형(스트레이트/해설/인터뷰 등) 
  - 글 분량 및 구조(단어 수, 단락 수, 구조적 특성) 
- **종합 평가 정보** 
  - 종합 품질 점수(0점 만점, 문제 발견시 마다 감점, -점수 부여) 
  - 품질 등급(신호등 표시 등 다양한 방식) 
  - 한 줄 평가 요약 
  - 핵심 강점 및 약점 요약(3줄 이내) 
- **차원별 세부 평가** 
  - 8개 평가 차원별 점수 및 핵심 피드백 
  - 문제 패턴 발견 빈도 및 심각도 
  - 발견된 문제와 관련 언론 규범/윤리 지침 연결 
- **교육적 요소** 
  - 미디어 리터러시 관련 팁 
  - 각 평가 기준에 대한 간략한 설명 
  - 관련 개념 용어집(glossary) 
- **비교 분석 정보** 
  - 동일 주제 타 매체 기사와의 품질 비교 
  - 해당 언론사의 시간 경과에 따른 품질 변화 추이 
  - 유사 주제 기사의 모범 사례 참조 
## **10-2-2. 시각화 전략** 
### **정보 계층화 구조**
기사 품질 평가 결과를 사용자에게 효과적으로 전달하기 위한 다층적 시각화 전략을 적용: 

- **Tier 1 (즉시 표시)** : 종합 품질 점수와 핵심 1-3개 문제점 
- **Tier 2 (탭/섹션으로 구분)** : 8개 평가 차원별 점수와 주요 이슈 
- **Tier 3 (클릭/확장 시 표시)** : 30개 세부 패턴 분석 결과 
- 인지 부하 이론(Cognitive Load Theory)에 기반한 접근으로, 사용자가 한 번에 처리할 수 있는 정보량을 고려한 설계 
### **소셜 미디어 최적화 시각화 요소**
- **'신뢰도 배지(Credibility Badge)'** : A+부터 F까지 학점 형태로 표시 
- 언론사 로고, 기사 제목 일부, 평가 날짜를 포함해 맥락 정보 제공 
- 배지 색상은 점수에 따라 차별화(A+/A: 녹색, B+/B: 파란색, C+/C: 노란색, D+/D: 주황색, F: 빨간색) 
- **'문제 패턴 아이콘'** : 주요 문제 패턴을 직관적인 아이콘으로 표시 
- **'비교 맥락 그래프'** : 해당 기사의 평가 점수를 동일 언론사 평균, 해당 주제 기사들의 평균과 비교하는 간단한 바 그래프 
- QR 코드나 단축 URL을 카드에 포함시켜 전체 분석 리포트 페이지로 쉽게 이동 가능 
### **설명 가능한 AI(XAI) 요소 통합**
- 특정 평가의 근거를 간결하게 제시 \
  "이 기사는 5개의 무주체 피동형 표현을 포함하고 있어 투명성 점수가 낮습니다" 
- 복잡한 알고리즘적 판단을 자연어로 변환하여 제시 
- 신뢰도 지표 함께 표시 (예: "이 분석의 확신도: 85%") 
- 레이더 차트로 8개 평가 차원 점수 시각화 
- 문제적 표현에 색상 코드 하이라이트 적용 및 마우스 오버 시 툴팁으로 문제점 설명 
## **10-2-3. 사용자 친화적 개선 제안** 
### **"Before/After" 방식**
- 문제 표현과 개선된 표현 병행 제시 \
  예: "최근 관측된다" → "연구팀이 최근 관측했다" 
- 맥락 중심 하이라이트: 문제 표현을 포함한 문장/문단 단위로 표시 
- 문제 유형별 시각적 코드화: 유형별 다른 색상과 아이콘 사용 
### **단계적 설명 구조**
1. 무엇이 문제인지 명시 
2. 왜 문제인지 저널리즘 원칙 연결 설명 
3. 어떻게 개선할 수 있는지 대안 제시 
4. 이러한 개선이 왜 중요한지 사회적 영향 설명 
### **계층적 정보 설계 기반 시각화**
평가 결과의 효과적인 시각화를 위해 계층적 정보 설계(Layered Information Architecture)를 적용한다: 

- **1단계(상단)** : 직관적인 종합 점수를 시각화. 100점 만점의 큰 숫자와 함께 A+부터 F까지의 등급을 컬러 코딩(녹색-노란색-빨간색)으로 표시하여 즉각적인 품질 인식이 가능하게 함(선택 사항). 
- **2단계(중단)** : 8개 평가 차원(진실성, 투명성 등)을 레이더 차트(Spider Chart)로 표현합니다. 각 차원에 마우스를 올리면 간단한 설명과 점수가 표시됨. 
- **3단계(하단)** : 원문 기사에 인터랙티브 하이라이트를 적용. 문제적 표현(무주체 피동형, 익명 취재원 등)에 색상 코딩된 밑줄을 그어 시각적으로 표시하고, 사용자가 클릭하면 다음 정보가 포함된 툴팁이 나타난다(저작권 문제로 기사원문을 그대로 시각화하기 어려울 수도 있음): 
  - 문제 유형(예: "무주체 피동형 표현") 
  - 영향받는 평가 차원(예: "투명성 -20점, 진실성 -15점") 
  - 개선 제안(예: "출처를 명확히 명시하세요") 
  - 추가 학습 자료 링크 

# **10-3. 서비스 안정성 및 확장성** 
## **10-3-1. 에러 처리 & 예외 상황 관리** 
### **입력 에러 처리**
- 잘못된 URL → "유효하지 않은 주소입니다. URL을 확인해주세요." 
- 텍스트 부족 → "기사가 최소 300자 이상이어야 분석이 가능합니다." 
- 파일 형식 오류 → "지원되지 않는 파일 형식입니다. PDF, DOCX, TXT 파일만 가능합니다." 
- 파일 크기 초과 → "파일 크기는 10MB 이하여야 합니다." 
### **서버 과부하 관리**
- 실시간 대기열 시스템: "현재 분석 대기 중입니다. 예상 소요 시간: 약 30초" 
- 단계별 진행 상황 표시: "기사 추출 중(25%)... 분석 중(50%)..." 
- 임계값 초과 시: "현재 사용자가 많아 서비스 속도가 지연되고 있습니다. 잠시 후 다시 시도해주세요." 
### **개인정보 보호 조치**
- 업로드된 문서 보관 정책: "분석 완료 후 24시간 내 모든 원본 데이터 자동 삭제" 
- 데이터 암호화 저장 및 접근 제한: "모든 사용자 데이터는 암호화되어 안전하게 보관됩니다." 
- 개인정보 최소화: 필수적인 정보만 수집하고 분석 결과와 연동 
### **시스템 장애 대응**
- API 연결 실패: "외부 서비스 연결에 실패했습니다. 잠시 후 다시 시도해주세요." 
- 분석 엔진 오류: "분석 과정에서 오류가 발생했습니다. 지원팀에 자동으로 보고되었습니다." 
- 시스템 복구: 자동 오류 감지 및 로깅 시스템으로 신속한 문제 해결 
## **10-3-2. 기술적 구현 고려사항** 
### **백엔드 처리 최적화**
- 병렬 처리로 여러 분석 모듈 동시 실행 
- 캐싱 시스템으로 자주 분석되는 URL 결과 빠르게 제공 
- 분석 대기열 관리로 트래픽 폭주 시에도 안정적 운영 
### **프론트엔드 사용성**
- 모바일 퍼스트 반응형 디자인으로 다양한 기기 지원 
- 단계별 진행 표시로 사용자 불안감 해소 
- 인터랙티브 시각화로 직관적 결과 이해 지원 
### **확장성 고려 및 미래 확장 로드맵**
- 모듈식 아키텍처로 새로운 평가 지표 추가 용이 
- API 제공으로 외부 서비스 연동 가능 
- 다국어 지원 준비 (영어, 일본어 등) 
- 브라우저 확장 프로그램 개발 및 대량 기사 분석 기능 추가 준비 
## **10-3-3. 커뮤니티 참여 및 확산 메커니즘** 
### **결과 공유 에코시스템**
- **맞춤형 공유 카드** : 평가 결과를 시각적으로 요약한 이미지 카드 자동 생성

- **종합 평가 정보** 
  - 종합 품질 점수(0점 만점, 문제 발견시 마다 감점, -점수 부여)

  - 한 줄 평가 요약

- **차원별 세부 평가** 
  - 8개 평가 차원별 점수 및 핵심 피드백 
  - 문제 패턴 발견 빈도 및 심각도

- **교육적 요소** 
  - 미디어 리터러시 관련 팁 
  - 각 평가 기준에 대한 간략한 설명 
  - 관련 개념 용어집(glossary) 
- **비교 분석 정보** 
  - 동일 주제 타 매체 기사와의 품질 비교 
  - 해당 언론사의 시간 경과에 따른 품질 변화 추이 
  - 유사 주제 기사의 모범 사례 참조
