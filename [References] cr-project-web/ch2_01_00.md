# **1. 로드맵**
1-1, 1-2 로드맵은 일종의 가이드입니다. 하지만 ‘단 하나’의 길이 아닌, 여러 길 ‘중 하나’에 대한 가이드일 뿐입니다. 

여러분이 가고 싶은 길을 떠올리고 각자의 로드맵을 그려 보세요. 

-----
\### 시나리오 1: 프롬프트 템플릿을 활용해 일단 한발 내딛기만 해도···

*"우리는 완성된 웹앱 형태의 플랫폼을 목표로 하지 않아요. 프롬프트 템플릿만으로 게릴라식으로 기사를 평가하고 결과를 공유하며, 문제의식을 확산시키려고 해요.”*

이런 접근법이라면? 몇 달이 아닌, 단 며칠 만에도 유의미한 반향을 만들 수 있습니다. 포털에 올라온 불량 기사를 GPT나 클로드 프롬프트에 넣고(+ ‘평가 템플릿.md’ 파일을 첨부) 평가한 후, 그 결과를 SNS에 공유하는 것만으로도 의미 있는 변화가 생길 수 있습니다. 

-----
\### 시나리오 2: 특정 분야에 집중한다면···

*"우리는 여론조사 보도에만 집중하려고 해요. 표본의 대표성, 오차범위 등 여론조사의 방법론적 한계를 제대로 설명하는지, 결과를 과장하거나 왜곡하지는 않는지, '[*선거여론조사보도준칙*](https://www.journalist.or.kr/news/section4.html?p_num=13)'의 조항들을 잘 지키고 있는지 평가할 거에요. (가능하다면) 설문조사 원본을 찾아 문항설계는 공정한지, 어떤 의도가 숨어있지는 않은지 등도 꼼꼼히 따져 보려고 해요.”*

분야를 좁히면 더 깊이 있게 분석할 수 있어요. 환경, 교육, 노동 등 특정 분야에 전문성을 가진 분들이 모여 해당 분야 보도의 품질을 평가하는 작은 프로젝트를 시작할 수 있습니다. 

-----
\### 시나리오 3: 더 근본적인 변화를 원한다면···

*"우리는 청소년들에게 미디어 리터러시를 가르치는 교육 프로그램을 운영하려고 해요. CR 프로젝트의 평가 기준을 활용해 학생들이 직접 뉴스를 분석하고 토론하는 워크숍을 진행할 계획입니다."*

아이들에게 비판적 읽기 능력을 키워주는 것은 언론 개혁의 근본 과제입니다. 교사, 청소년 지도사, 학부모 등이 함께할 수 있는 프로젝트입니다. 

-----
여러분 각자의 로드맵을 그려보세요. 우리가 제시하는 단계별 계획은 참고만 하시고, 여러분 각자의 상황과 목표에 맞게 자유롭게 재구성해 보세요. 중요한 건 일단 ‘한발 내딛는 것’입니다. 작은 실험이 모여 큰 변화를 만들어낼 수 있습니다. 

# **1-1. 기본 흐름** 
### **개요**
-----
언론개혁 필요성에 공감하는 시민들이 다양한 경로를 통해 '온라인 소통 공간'에 모임. 

시민들이 '뉴스 분석 방식', '기술적 조건'에 대해 논의하면서, "어떤 기준으로 기사 품질을 평가할지, 어떻게 기술적으로 구현할지' 등 관련 사항을 하나씩 합의. 

평가 기준과 기술적 구현 방식이 구체화되면, 언론·기술 분야 '전문가 시민'들이 참여해 '평가 템플릿 페이지'(기사 유형별로 다른 평가 템플릿 제공)를 마련. 이 페이지를 통해 수집된 평가 데이터로 오픈소스 언어모델을 학습시켜, 자체 평가 플랫폼을 구축. 캠페인을 통해 플랫폼을 널리 알리고, 순차적으로 '제3비평 그룹'에서 다른 유형(영상 뉴스, 칼럼/사설, 경제기사 전문, 선거보도 특화···)의 비평 플랫폼을 개발하도록 지원함. 연대와 확장을 거듭하며 기획 의도를 달성. 

정리하면 
### **0.5단계: 사전 준비** 
- 시민 + 언론윤리 체크 + 도구(AI 활용) 기본 컨셉트 제안. 
- 프로젝트 소개 페이지(cr-project.org)와 평가 템플릿 샘플 마련 
### **1단계: 평가 템플릿 웹페이지** 
- 온라인 소통 공간에서 '뉴스 평가 기준'과 '기술적 구현 방식'에 대해 논의 
- 즉시 실행 가능한 평가 템플릿 페이지(프롬프트 기반 개별 평가, 템플릿 파일 배포 & 피드백 수집) 마련 
### **1.5단계: 평가데이터 수집·자체 모델 학습** 
- 템플릿 사이트를 통해 수집된 평가데이터를 활용해 오픈소스 LLM 파인튜닝. 
- 자체 AI모델 기반 평가플랫폼 구상. 
### **2단계: 자체 평가플랫폼 구축** 
- 뉴스 분석 플랫폼 ≒ 언론윤리체크 도구, 공개(독립된 평가 플랫폼과 템플릿 기반 평가 사이트 운영을 병행하며 다양한 사용자가 활용할 수 있게함). 
- 법률·경영 전문가의 도움으로 시스템 현실화 
### **2.5단계: 캠페인 & 제3 비평 플랫폼 지원** 
- 캠페인을 통해 사용자 저변 확대, 시민 미디어 비평 운동으로 확산. 
- 다른 유형의 비평 플랫폼(다른 매체, 다른 방식) 개발을 독려하고 지원함. 연대를 통해 '시민 비평 네트워크'로 확장. 

# **1-2. 단계별 실행 계획** 
## **0.5단계: 사전 준비** 
### **기본 컨셉트 제안 및 기초 자료 정리** 
- 시민 + 언론윤리 체크 + 도구(AI 활용) 기본 컨셉트 제안 
- 저널리즘·뉴스 비평 관련 도서 및 연구 논문 정리 
- 평가 기준 초안 작성 
- 유사 프로젝트 확인 및 기술 스택 검토 
- 학습용 데이터 파악 
### **소개 페이지와 평가 템플릿 샘플 마련** 
- 프로젝트 소개 페이지(cr-project.org) 준비 
- 8개 평가 차원(진실성, 투명성 등)을 중심으로 문제적 보도 패턴(무주체 피동형, 익명 취재원 등) 정리 
- 프롬프트에 첨부할 '평가 템플릿.md'(문제적 보도 관행 + 언론윤리규범) 샘플 작성 
- 사용자가 직접 생성형AI챗봇에 평가 지시문을 입력해 기사를 평가할 수 있게 가이드 작성 
### **시민 그룹 피드백** 
- 시민사회단체(언론 대응이 필요할 법한), 독립언론에 프로젝트를 소개하고 의견을 구함. (취지에는 공감하나, 적극적으로 활용하려는 의지는 '글쎄'. 활동을 악의적으로 왜곡하는 경우는 극히 드물고, 대체로 단체 의제를 다뤄주는 매체와 호혜 관계)
- 언론, 경제, 사회, 교육, 노동, IT 등 분야 전문가들에게 조언을 구하고 참여를 요청 
- 일부 커뮤니티에 프로젝트를 소개하고, 시민 참여 가능성 확인(프로젝트가 틀을 갖추고 구심점이 생기면 호응하는 시민이 늘어날 것)
- 미디어리터러시 기관/단체 등 기존 시스템을 활용한 시민비평 프로그램 제안(언론진흥재단 프로그램을 활용한 '언론윤리 서포터즈'를 제안했으나 정부 조직 개편 등이 선행되어야 다뤄질 수 있을 것)
## **1단계: 평가 템플릿 웹페이지** 
### **온라인 소통 공간에서 '뉴스 평가 기준'과 '기술적 구현 방식'에 대해 논의** 
- ‘단계별 실행 계획’ 전면 재설정 
- 문제적 보도 관행 사례 수집 및 유형화(계속 업데이트)
- 평가 대상과 방식에 대한 합의(매체와 내용에 따라 평가 대상으로 삼을지, 점수화할 것인지, 언론윤리규범 매칭 수준을 어느 정도로 잡을지, 평가 리포트를 어떻게 구성할지 등)
- 기사 유형에 맞게 평가 기준을 세분화해 마련 
### **평가 템플릿 페이지(프롬프트 기반 개별 평가, 템플릿 파일 배포 & 피드백 수집) 마련** 
- 기사 유형별 평가 템플릿 마련(정치, 경제, 사회, 지역, 국제 등). 사용자가 생성형 AI 프롬프트에 평가 지시문을 입력해 직접 기사를 평가할 수 있게 안내. 뉴스/기사를 직접 평가하고 품질을 확인하면서 효능감을 느낄 수 있게 설계 
- 템플릿 파일 배포 웹페이지 구성(선택 마법사(Wizard) 방식으로 기사 유형에 맞는 템플릿 제공)
- 평가 결과를 공유하고, 토론할 수 있는 커뮤니티 플랫폼 마련 
- 소규모 테스트 그룹 운영. 동일 기사에 대한 다양한 AI 모델의 평가 결과 비교 분석 
- 프롬프트 성능 및 사용자 경험 피드백 수집. 평가 결과를 구글폼으로 제출하게 유도(Make, n8n 등 자동화도구 활용)
- 피드백 의견을 반영해 프롬프트 최적화하고, 사용자의 AI서비스에 맞는 모델별 최적화 버전 업데이트 
## **1.5단계: 평가데이터 수집·자체 모델 학습** 
### **수집된 평가데이터를 활용해 오픈소스 LLM 파인튜닝** 
- 프롬프트 기반 시스템에서 수집된 누적 평가 데이터 분석(계속 업데이트)
- 정식 AI 모델 학습용 데이터셋으로 가공 
### **오픈소스 LLM 기반 자체 평가플랫폼 구상** 
- 오픈소스 모델 성능 평가, 한국어 뉴스 평가에 적합한 모델 선정 
- 정식 플랫폼 요구사항 구체화 
- 시스템 아키텍처 초안 설계 & 선택된 오픈소스 모델 파인튜닝 
- 외부 전문가 검증 후 MVP 구축, 베타 테스트 진행 
- 모델의 편향성 검증 및 윤리적 평가 수행 
## **2단계: 자체 평가플랫폼 구축** 
### **뉴스 분석 플랫폼 ≒ 언론윤리체크 도구 공개(독립된 평가 플랫폼과 템플릿 기반 평가 사이트 운영을 병행하며 다양한 사용자가 활용할 수 있게함)**
- 독립적인 & 안정적인 플랫폼(백엔드/프론트엔드/데이터베이스) 구축 
- 평가 결과를 쉽게 공유할 수 있게 UI/UX 요소 설계(가령 공유하고 싶은 부분을 드래그하면 "내용 공유하기" 팝업이 표시되거나, 해당 부분이 SNS 최적화 이미지로 자동 변환되는 방식)
- 프롬프트 템플릿 기반 사이트와 병행 운영. 상황과 필요에 따른 선택적 접근 가능. (시민들이 ChatGPT나 Claude 같은 기존 AI 서비스에 프롬프트 템플릿을 직접 입력해서 기사를 평가하는 분산형 시스템이 작동하고, 다른 한쪽에서는 이렇게 수집된 평가 데이터를 활용해 CR만의 전용 AI 모델을 학습시키는 중앙집중형 시스템이 동시에 운영되는 구조. 데이터의 순환 구조에 주목. 시민들이 분산형 프롬프트 시스템으로 생성한 평가 결과가 중앙 데이터베이스로 수집되어 AI 모델 학습에 활용되고, 개선된 AI 모델의 성능이 다시 프롬프트 템플릿의 정교화에 반영되는 선순환 체계를 구축하는 것. 이를 통해 초기에는 진입 장벽이 낮은 프롬프트 방식으로 시민 참여를 유도하면서도, 장기적으로는 독립적이고 정교한 평가 플랫폼으로 발전시킬 수 있는 기술적 기반을 마련할 수 있음.)
- 신규 사용자가 플랫폼의 기능과 활용법을 쉽게 익힐 수 있도록 단계별 튜토리얼, 동영상 가이드, 도움말 섹션 마련 
### **법률·경영 전문가의 도움으로 시스템 현실화** 
- 이용약관, 개인정보처리방침, 면책 조항 등 법률적 안전장치 마련. 기사 인용 최소화, 원문 링크 및 변형적 활용으로 저작권 리스크를 완화하고 있음을 공지하고, 이용자들에게 법적으로 안전한 평가 공유 방식을 소개, 피해야 할 핵심 표현 목록 등을 정리해 알림. 추가로 객관적 사실 중심의 평가 체계 및 이의제기 절차를 담은 가이드라인도 작성 
- 개발 및 운영 비용을 산출하고, 비영리 운영 모델을 전제로 펀딩이나 후원을 통해 조화롭게 비용을 마련할 수 있을지 경영 자문을 구함 
## **2.5 단계: 캠페인 & 제3 비평 플랫폼 지원** 
### **기술 관리 및 확장** 
- 시스템 안정화. 지속적인 프롬프트 개선 및 LLM 업그레이드 
- 특수 분야, 상황별로 특화된 프롬프트 추가 개발(예를 들어, 지역언론을 위한 윤리규범 & 평가 템플릿 작업)
