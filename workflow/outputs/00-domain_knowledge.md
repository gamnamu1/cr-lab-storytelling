# 도메인 지식: 언론/탐사보도 + IT/개발자 문화

---

## 1. 이 세계의 실제 작동 방식

### 언론계: 공식 vs 실제

**공식적으로 말하는 것:**
- "우리는 공정하고 객관적인 보도를 추구합니다"
- "언론의 독립성과 공익성을 지킵니다"
- "언론윤리강령을 준수합니다"

**실제로 작동하는 방식:**
- 광고주(대기업, 정부)에 불리한 기사는 자연스럽게 누락된다
- '투 소스 룰'(두 개 이상의 독립적 취재원 확인)은 마감 압박 앞에서 무력화
- 보도자료를 거의 그대로 옮기는 '보도자료 저널리즘'이 기사 생산의 상당 부분 차지
- 온라인 기사의 몰래 수정—정정 공지 없이 슬쩍 고치는 관행
- 클릭 수, PV(페이지뷰)가 기자 평가의 주요 지표가 됨

### IT/개발자 세계: 공식 vs 실제

**공식적으로 말하는 것:**
- "기술은 도구일 뿐, 중립적입니다"
- "우리는 코드로 문제를 해결합니다"
- "오픈소스 정신으로 공유합니다"

**실제로 작동하는 방식:**
- 알고리즘 설계 시 '자극적인 콘텐츠가 더 많이 노출되도록' 하는 선택이 암묵적으로 이루어짐
- 제품 결정에서 "사용자 경험"보다 "매출/전환율"이 우선될 때가 많음
- 기술적 결정의 사회적 영향에 대한 논의는 "정치적"이라며 회피됨
- 시빅해킹 커뮤니티는 존재하지만, 대다수 개발자는 "본업"과 분리된 취미로 인식

### 두 세계가 만나는 지점

시빅해킹/시빅테크는 기술로 공공 문제를 해결하려는 움직임이다.
- 코로나 시기 '공적마스크 앱'이 대표 사례—정부 데이터를 시민이 활용할 수 있게 함
- 언론 관련 시빅테크는 더 민감—"언론 평가"는 곧 정파성 논란으로 번질 수 있음
- 개발자들은 "내 기술이 정치적으로 해석될 수 있다"는 사실에 불편함을 느낌

## 2. 이 세계의 사람들

### 언론계 인물 유형

**1. 이상주의 탐사기자**
- 동기: 진실 추적, 사회 변화에 대한 믿음
- 가치관: 저널리즘 본질에 대한 강한 신념
- 내적 갈등: 점점 작아지는 영향력, 조직 내 고립, 번아웃
- 말투: 냉소와 열정이 섞인 어조

**2. 현실적 생존자**
- 동기: 가족 부양, 경력 유지
- 가치관: "이 정도면 양심 팔지 않은 거야"
- 내적 갈등: 자기 합리화와 자괴감 사이
- 말투: "세상이 원래 그래"

**3. 편집국 정치인**
- 동기: 승진, 권력, 인정
- 가치관: 조직 내 생존이 최우선
- 내적 갈등: 과거의 자신과 현재의 타협 사이
- 말투: 모호하게, 책임 회피하며

**4. 언론 비평가/활동가**
- 동기: 언론 개혁, 미디어 리터러시
- 가치관: 시민의 알 권리
- 내적 갈등: 목소리는 높지만 실제 변화는 더딤, 정파적으로 비칠 위험
- 말투: 분석적, 때로는 분노가 섞인

### IT/개발자 인물 유형

**5. 순수 기술주의자**
- 동기: 기술적 우아함, 문제 해결의 쾌감
- 가치관: "좋은 코드가 좋은 세상을 만든다"
- 내적 갈등: 기술의 사회적 영향에 대한 질문이 불편함
- 말투: 기술 용어 중심, 정치적 화제 회피

**6. 시빅해커**
- 동기: 기술로 사회 문제 해결
- 가치관: 기술의 공공성
- 내적 갈등: 본업과의 균형, 지속가능성
- 말투: 열정적이지만 때로 이상주의적

**7. 회의적 중립자** (주인공 유형)
- 동기: 리스크 회피, 평온한 삶
- 가치관: "기술은 중립적, 정치에 관여하지 않겠다"
- 내적 갈등: 회피가 곧 선택이라는 인식과 불편함
- 말투: 방어적, 논리적 정당화

## 3. 드라마의 씨앗들

### 딜레마 1: 중립의 불가능성
- 추천 알고리즘에서 "어떤 콘텐츠를 먼저 보여줄까"는 이미 판단
- 뉴스 평가 플랫폼에서 "어떤 기준으로 평가할까"도 이미 선택
- "아무것도 안 하겠다"는 결정도 현 상태 유지에 기여

### 딜레마 2: 기술자의 책임 범위
- 총을 만든 사람은 총으로 죽인 사람에게 책임이 있는가?
- 알고리즘을 설계한 사람은 그 알고리즘이 유발한 중독에 책임이 있는가?
- 어디까지가 "내 책임"인가?

### 딜레마 3: 가까운 관계 vs 신념
- 연인/친구가 과거 유사 프로젝트로 피해를 입었다면?
- "당신이 참여하면 나는 힘들어"라는 말 앞에서 어떻게 할 것인가?
- 개인의 선택이 관계에 파장을 미칠 때

### 딜레마 4: 직업적 안전 vs 시민적 행동
- 클라이언트 중 언론사가 있다면, 언론 평가 프로젝트 참여는 자살행위?
- 조용히 살 것인가, 목소리를 낼 것인가?
- 불이익을 감수할 준비가 되어 있는가?

### 딜레마 5: 완벽한 해답의 부재
- "언론윤리규범을 기준으로"—그 규범도 완벽하지 않다
- 기술적으로 뉴스 품질을 평가한다는 것 자체의 한계
- 그래도 "최소한 정직하게" 시도할 가치가 있는가?

## 4. 감각적 디테일

### 언론사 편집국
- **공간**: 형광등 불빛, 모니터 여러 대, 벽면의 TV(24시간 뉴스 채널), 흩어진 종이
- **소리**: 키보드 소리, 전화 통화, 급한 목소리 "마감 몇 분 남았어!"
- **시간**: 오전 회의, 오후 취재, 저녁 마감—야근은 일상
- **언어**: "짤" (사진), "앵글" (기사 방향), "물었다" (질문하다), "데스크" (상급자)

### 개발자 환경
- **공간**: 듀얼/트리플 모니터, 스탠딩 데스크, 헤드폰, 커피
- **소리**: 기계식 키보드 타각음, 슬랙 알림음, 간헐적 화상회의
- **시간**: 플렉스 타임이지만 밤샘 배포는 어쩔 수 없음
- **언어**: "머지하다" (병합), "디플로이" (배포), "레거시" (낡은 시스템), "기술 부채"

### 시빅해킹 밋업
- **공간**: 공유 오피스, 화이트보드, 포스트잇, 노트북 열어놓은 사람들
- **소리**: 캐주얼한 토론, 웃음, "이거 해커톤에서 만들었는데..."
- **시간**: 주로 저녁/주말—"본업 외 시간"
- **언어**: "임팩트" (사회적 영향), "오픈데이터" (공개 데이터), "MVP" (최소 기능 제품)

## 5. 클리셰 vs 현실

### 언론 관련

| 클리셰 | 현실 |
|--------|------|
| 기자는 모두 진실을 위해 싸운다 | 대다수는 마감에 쫓기는 생활인 |
| 편집국은 치열한 토론의 장 | 위계적 지시와 암묵적 자기검열 |
| 특종은 영웅적 기자의 산물 | 대부분 제보에서 시작, 팀 작업 |
| 언론사는 정파적으로 양분 | 경제적 이해관계가 더 결정적일 때 많음 |

### IT/개발자 관련

| 클리셰 | 현실 |
|--------|------|
| 개발자는 세상을 바꾸는 영웅 | 대부분은 티켓 처리하는 일상 |
| 기술은 순수하고 중립적 | 설계 단계부터 가치가 개입 |
| 시빅해킹은 순수한 봉사 | 커리어 개발, 네트워킹 동기도 존재 |
| 알고리즘은 객관적 | 학습 데이터와 설계자의 편향 반영 |

## 6. 다중 관점 지도

### 체제 내부자의 논리

**언론계:**
- "마감 압박 속에서 완벽한 검증은 불가능하다"
- "광고 수익 없이는 저널리즘 자체가 불가능하다"
- "독자들이 자극적인 제목을 클릭하니까 우리도 어쩔 수 없다"
- "외부 비판은 현장을 모르는 소리"

**IT계:**
- "기술은 도구일 뿐, 사용자가 어떻게 쓰는지는 우리 통제 밖"
- "회사가 원하는 걸 만드는 게 내 일이다"
- "정치적 논쟁에 휘말리면 본업에 지장"
- "완벽한 시스템은 없다, 우리는 최선을 다했다"

### 체제 비판자의 논리

**언론 비판:**
- "언론이 스스로 만든 윤리강령조차 안 지킨다"
- "광고 모델 자체가 저널리즘을 왜곡한다"
- "뉴스 생산 구조가 바뀌지 않으면 콘텐츠도 안 바뀐다"
- "시민의 알 권리가 상업적 이익보다 우선해야 한다"

**기술 비판:**
- "중립성 주장은 책임 회피의 핑계"
- "설계 단계부터 윤리적 고려가 필요하다"
- "기술자도 자신이 만드는 것의 사회적 영향을 인식해야 한다"
- "참여하지 않는 것도 선택이며, 그 선택에도 책임이 따른다"

### 중간 지대

- 조직 안에서 최소한의 양심을 지키려 노력하는 기자
- "내가 할 수 있는 범위 내에서" 개선하려는 개발자
- 급진적 변화보다 점진적 개선을 믿는 현실주의자
- 이상과 생존 사이에서 타협점을 찾는 사람들

### 외부자/피해자의 시선

**언론 피해자:**
- 오보로 삶이 망가진 사람들
- "기레기"라는 멸칭이 일상화된 대중의 불신
- "언론은 다 똑같아"라는 냉소

**기술 피해자:**
- 추천 알고리즘에 중독된 청소년의 부모
- 자극적 콘텐츠만 보게 된 사용자
- "내 데이터가 어떻게 쓰이는지 모르겠다"는 불안

## 소설적 활용 제안

### 개발자 주인공의 내면 갈등

```
그는 모니터 앞에 앉아 있었다. 추천 알고리즘의 precision 수치가
0.89에서 0.92로 올랐다. 좋은 숫자다. 그런데 문득 떠오른
피드백—"아이가 이상한 영상만 봐요"—을 떨쳐낼 수 없었다.
나는 확률만 계산했을 뿐이야. 그렇게 생각하려 했다.
```

### 시빅해킹 제안 장면

```
"공적마스크 앱도 중립적이었나요?"
그 질문에 그는 대답하지 못했다. 정부는 재고 공개를 원하지 않았다.
개발자들이 나서서 '시민은 정보에 접근할 권리가 있다'고 주장했다.
그건 기술적 판단이 아니었다. 정치적 판단이었다.
```

### 가까운 사람과의 갈등

```
"그 프로젝트, 하지 마."
그녀의 목소리는 단호했다. 3년 전, 비슷한 플랫폼 때문에
그녀는 몇 달간 악플에 시달렸다. 신상이 털렸다.
"기술은 중립적이라고? 그 중립적인 기술 때문에 내 삶이 어떻게 됐는지 알아?"
```
